# 02: Bugs, Style, and Documentation

## 1. Bugs and Errors
- **Bugs** are mistakes in code. If you forget a closing parenthesis `)`, the interpreter throws a Syntax Error.
- Error messages are your best friend. They tell you the exact line and type of error.

## 2. Comments
- Comments (`#`) are notes for yourself and other developers. They track intentions and explain *why* a block of code exists.
  ```python
  # Ask the user for their name (Intention)
  name = input("What's your name? ")
  ```

## 3. Pseudocode
- **Pseudocode** is a special type of comment used to outline the logic of a program *before* writing actual Python code. It acts as a to-do list for complex algorithms.

## 4. Code Style (PEP 8)
- **Style** refers to how readable and consistent your code looks. Is your indentation correct? Do your variables have clear names? Are there appropriate blank lines?
- Python's official style guide is called **PEP 8** (Python Enhancement Proposal 8). Following standard style guidelines is critical when working in teams, as it ensures everyone's code looks uniform and is easy to review.
